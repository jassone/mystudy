## 一、cpu

### 1、CPU Usage By Cores(%)

表示每个CPU核心的使用率。在多核心处理器系统中，每个核心都可以独立执行任务。因此，查看每个核心的使用率可以帮助你了解系统的负载分布情况。例如，如果所有核心的使用率都很高，那么可能需要考虑升级硬件或优化代码以提高性能。

如果指标是 10，那么这通常表示当前 CPU 的 10% 被使用。这个指标是一个**百分比**，用于表示 CPU 的使用程度。如果你的系统有多个核心，这个数字可能是所有核心使用率的平均值，或者是单个核心的使用率，具体取决于你的监控工具的设置。



### 2、CPU Usage(%)

CPU 使用率（CPU Usage）是一个衡量计算机处理器（CPU）使用程度的指标。它表示在特定时间段内，CPU 在执行指令（包括用户程序和系统程序）上花费的时间占总时间的**百分比**。如果 CPU 使用率接近或达到 100%，则表示 CPU 处于高负载状态，可能会导致系统响应缓慢。



### 3、CPU User Time(%)

是一个性能指标，表示 CPU 时间中有多少百分比被用户级应用程序使用。这包括执行用户进程的时间，但不包括系统在执行内核代码时所花费的时间。这是衡量 CPU 被应用程序使用程度的一种方式。如果这个值很高，可能表示你的应用程序正在密集地使用 CPU。





## 二、内存

### 1、RSS

RSS（Resident Set Size）：这是进程当前使用的物理内存量，包括所有堆栈、堆和共享库的大小。但是，这并不包括交换出去的内存。RSS是一个有用的指标，因为它可以帮助我们理解进程正在使用多少物理内存。

### 2、WSS

WSS（Working Set Size）：这是在特定时间窗口内实际使用的内存量。换句话说，这是进程在特定时间窗口内实际需要的内存量。这个指标可以帮助我们理解进程的内存使用模式，并可以帮助我们预测在未来的内存需求。

### 1、Average System Load

是一个衡量计算机系统负载的指标。它表示在一定时间内，系统中处于运行状态或等待运行的进程平均数量。这个指标可以帮助我们了解系统的负载情况，如果这个值过高，可能表示系统过于繁忙，需要进行优化或升级。

如果值是 0.6，这意味着在过去的一段时间内（通常是最近1分钟、5分钟或15分钟，具体取决于你的系统和工具），系统中平均每时刻有0.6个进程处于运行状态或等待运行。

这个值是一个相对的指标，其解读取决于你的系统有多少个处理器核心。例如，如果你的系统有一个处理器核心，那么0.6的系统负载意味着你的系统在大约60%的时间里有进程在运行或等待运行，其余40%的时间系统是空闲的。如果你的系统有两个处理器核心，那么0.6的系统负载意味着你的系统在大约30%的时间里有进程在运行或等待运行，其余70%的时间系统是空闲的。

总的来说，如果 "Average System Load" 的值小于你的处理器核心数量，那么你的系统应该是有足够的资源来处理当前的工作负载的。如果这个值大于你的处理器核心数量，那么可能有一些进程需要等待处理器资源，你的系统可能会感到繁忙或者延迟。
